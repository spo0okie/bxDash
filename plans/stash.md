# Долгий ящик

## Цель

В приложении есть такая секция для задач как "долгий ящик" - задачи без крайнего срока, либо с крайним сроком больше, чем граница отображаемого календаря.
Нужно разделить долгий ящик на три (в порядке убивания приоритета) по статусу задачи: ожидает выполнения, новая (бэклог), отложено.

### UI layout

Новый режим отображения должен быть опциональным, чтобы можно было переключаться между ним и старым режимом, в котором все задачи без крайнего срока отображаются в одном "долгом ящике".

#### Долгий ящик внизу

```text
Сетка: X - пользователи, Y - время (вниз от прошлого к будущему)
Каждая ячейка - неделя для пользователя

    ┌─────────┬─────────┬─────────┬─────────┐
    │  User1  │  User2  │  User3  │  User4  │  ← пользователи (ось X)
    ├─────────┼─────────┼─────────┼─────────┤
    │ Неделя1 │ Неделя1 │ Неделя1 │ Неделя1 │
    ├─────────┼─────────┼─────────┼─────────┤
    │ Неделя2 │ Неделя2 │ Неделя2 │ Неделя2 │  ← время (ось Y, вниз)
    ├─────────┼─────────┼─────────┼─────────┤
    │ Неделя3 │ Неделя3 │ Неделя3 │ Неделя3 │
    ├─────────┼─────────┼─────────┼─────────┤
    │  User1  │  User2  │  User3  │  User4  │
    │ Ожидает │ Ожидает │ Ожидает │ Ожидает │  ←┐
    ├─────────┼─────────┼─────────┼─────────┤   │
    │  User1  │  User2  │  User3  │  User4  │   │
    │  Новые  │  Новые  │  Новые  │  Новые  │  ←┼ Элементы без назначенной даты
    ├─────────┼─────────┼─────────┼─────────┤   │ либо с датой за пределами календаря
    │  User1  │  User2  │  User3  │  User4  │   │
    │ Отложено│ Отложено│ Отложено│ Отложено│  ←┘
    └─────────┴─────────┴─────────┴─────────┘
```
